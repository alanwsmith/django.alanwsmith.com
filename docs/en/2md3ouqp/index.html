Section 4 - Adding Logout Functionality

Introduction

We've got the ability to log in. Now we need to be able to log out. We'll do that in this section.

Make sure you've activated your venv and started the server if it's not already running with:

terminal commands 
python manage.py runserver

Section Steps

  1. Add the logout form to the home page

    We're using a standard <a> HTML tag for our login link. We have to handle the logout functionality a little differently. For security reasons, Django requires that we use a form so that we can add CSRF protection.

    The basic code for the form is:

    logout form code
    <form action="{% url 'logout' %}" method="post">
  {% csrf_token %}
  <button type="submit">Log Out</button>
</form>

    The action attribute defines the URL the form is sent to when submitted. In this case, we're using this url template tag

    logout template tag
    {% url 'logout' %}

    The built-in user auth functionality provides logout for us automatically. And, unlike the login link from the prior section we don't need to make a template for logout. That's because Django will automatically redirect the user to another page after the log out process runs.

    The form also contains our CSRF security token.

    csrf template tag 
    {% csrf_token %}

    We don't need to so anything with it other than making sure it's inside our form on the page.

    Update the tutorial_site/homepage/templates/homepage/index.html template with this code to add the form:

    update for: tutorial_site/homepage/templates/homepage/index.html
    <!DOCTYPE html>
<html>
  <body>

    <h1>Hello, world</h1>

    <div>
      <a href="{% url 'login' %}">Login</a>
      <form action="{% url 'logout' %}" method="post">
        {% csrf_token %}
        <button type="submit">Log Out</button>
      </form>
    </div>

  </body>
</html>

    Start the server if it's not already running with:

    terminal commands 
    python manage.py runserver

    Then visit the home page to see the logout button:

    home page link

    http://127.0.0.1/ ↗

  2. Redirecting logout to the home page

    Django's default behavior when you click the Log Out button is to redirect you to a page that shows a "Logged out" message in a "Django administration" page template. That's not bad in and of itself, but it's most likely not what we'd want on a full site. So, let's set the logout to redirect to the home page instead.

    All we need to do to make that happen is to add this line to the end of the tutorial_site/settings.py file (after the DEFAULT_AUTO_FIELD line):

    append to: tutorial_site/settings.py
    LOGOUT_REDIRECT_URL = "/"

  3. Redirect login to the home page

    We can update the Log In link to redirect to the home page (instead of the user profile page) in the same way by adding this line after our new LOGOUT_REDIRECT_URL line:

    append to: tutorial_site/settings.py
    LOGIN_REDIRECT_URL = "/"

    Wrapping Up

    If we go to the home page now, both the Login link and the Log Out button work.

    home page link

    http://127.0.0.1/ ↗

    It's a little hard to see that though since we don't have any indication of if we're logged in or not. We'll address that next.

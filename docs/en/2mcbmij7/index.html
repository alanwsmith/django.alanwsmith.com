Section 2 - Make A Home Page

Introduction

Django provided us with a nice default home page to let us know that things are working, but we don't want it to be what folks would see if they visited our site. Instead, we'll make our own basic home page.

Before we get started, make sure the Python virtual environment is active .

Section Steps

  1. Examine the initial url paths

    The first change we're going make is to update the routing functionality to point calls to the root of the site. These changes are made in the file :

    file path: 
    tutorial_site/urls.py

    Before we make any changes, let's examine the initial version of the file that looks like this :

    original: tutorial_site/urls.py
    # A bunch of comments are here that we're 
# going to ignore

from django.contrib import admin
from django.urls import path

urlpatterns = [
    path("admin/", admin.site.urls),
]

    These first two lines of code are :

    python
    from django.contrib import admin
from django.urls import path

    These lines import some of Django's default functionality that's used for the configuration.

    The next snippet of code is :

    python
    urlpatterns = [
    path("admin/", admin.site.urls),
]

    The urlpatterns = [] list holds a collection of path() functions. These are what define the specific url paths that define the routes.

    The path() functions take two required arguments. The first is a pattern to match in the requested page's URL. The second is what to do when a match is found. (The path() function can also take two optional arguments for "kwargs" and "name" that we'll see later.)

    The initial path that's include by default is :

    python
    path("admin/", admin.site.urls)

    It tells Django to match "admin/" from the URL of a page request and send it to the built in admin functionality defined in admin.site.urls . This is what powers the http : //127.0.0.1 : 8000/admin/ pages.

  2. Adding the new url path route

    We need to do two things to create the new url path route for the home page. First, we need to import a new module with :

    python
    from django.urls import include

    Then, we need to add a new path with :

    python
    path("", include("homepage.urls")),

    The empty "" match argument means the path() is for the root of the site (i.e. http : //127.0.0.1 : 8000/) and the include("homepage.urls") tells Django that we want to use an app/module named "homepage" to handle it. (We haven't created this module yet, so this will cause an expected error that we see below.)

    Putting those two lines in place updates the file to look like this which you can copy and paste to replace the contents of the original :

    update for: tutorial_site/urls.py
    from django.contrib import admin
from django.urls import path
from django.urls import include

urlpatterns = [
    path("", include("homepage.urls")),
    path("admin/", admin.site.urls),
]

    When we save that file the Django server in the terminal will reload itself and show an error that ends with :

    command line error: 
    ModuleNotFoundError: No module named 'homepage'

    It'll take us a few steps to get the home page to show up, but we'll start by tackling that error.

  3. Stop the server

    We'll need to run another python manage.py command in the next step. Let's stop the server by holding down the control key on the keyboard then pressing c

    keyboard: 
    control + c

  4. Make the new app/module

    The error message starts with "ModuleNotFountError". In Django's terminology the module it's looking for is called an "app". (TODO : more details on app/module).

    Run this command to create our new "homepage" app/module :

    command line: 
    python manage.py startapp homepage

  5. Register the newly created app/module

    After creating the app/module with "startapp" in the previous command we have to configure Django to use it. We do this by adding this line :

    python
    "homepage.apps.HomepageConfig",

    to the INSTALLED_APPS list variable in the file :

    file path: 
    tutorial_site/settings.py

    There's a lot to that file so you'll have to scroll down some to find it. When you've made the update, the variable should look like this :

    python
    INSTALLED_APPS = [
    "django.contrib.admin",
    "django.contrib.auth",
    "django.contrib.contenttypes",
    "django.contrib.sessions",
    "django.contrib.messages",
    "django.contrib.staticfiles",
    "homepage.apps.HomepageConfig",
]

    TODO : Link to some docs about how the "homepage.apps.HomepageConfig", line works.

  6. Start the server

    With that change in place, we're ready to start the server again which will show us a new error. Run this to start the server :

    command line: 
    python manage.py runserver

    The error in the console changes from :

    command line error: 
    ModuleNotFoundError: No module named 'homepage'

    To :

    command line error: 
    ModuleNotFoundError: No module named 'homepage.urls'

  7. Stop the server

    We're going to add a new file in the next step. We'll need to restart the server to see the change so we can stop it now with :

    keyboard: 
    control + c

  8. Add a homepage/urls.py file

    Django is looking for a file at :

    file path: 
    homepage/urls.py

    When we ran the python manage.py startapp homepage command it built a bunch of default files in the "homepage" directory, but "url.py" wasn't one of them. We have to make it ourselves. Create it now and add this for the contents of the file :

    new file: homepage/urls.py
    from django.urls import path

from . import views

app_name = "homepage"
urlpatterns = [
    path("", views.index, name="index"),
]

    This file looks a lot like :

    file path: 
    tutorial_site/urls.py

    In fact, they two files work together. Inside the tutorial_site/urls.py file the line :

    python
    path("", include("homepage.urls")),

    calls out to include the file we just created at :

    file path: 
    homepage/urls.py

  9. Examine the new urls.py file

    Let's take a quick look at the lines of the new "urls.py" file to see what's going on.

    Details :

    • The first line is a the basic import statement that let's use use the path() function in our urlpatterns variable later in the file.

      python
      from django.urls import path

    • The next line of code is :

      python
      from . import views

      This imports the "views.py" file from the current directory. In this case that means it's pointing to :

      file path: 
      homepage/views.py

      We'll be editing that file in our next step.

    • The next code line is :

      python
      app_name = "homepage"

      The "app _ name" provides us with part of what we need to automatically generate links. We'll see more about how that works in a moment.

    • Next we get to :

      python
      urlpatterns = [
    path("", views.index, name="index"),
]

      These paths are combined with those from the top level file (i.e. tutorial_site/urls.py ) to create that pattern matches for requested URLs and assocaite them with their respective actions.

      In this case, we have an "" empty string for the matcher. TODO : show how this works with the top level urls.py file to serve the home page.

      The "views.index" parameter tells Django that we want to use the "index" function from the "views.py" file that was imported earlier. (That function doesn't exist yet, so we'll expect an error that we'll discuss in a moment)

      The last argument passed to the path is name="index" . This name is the second component we can use to automatically generate links to specific URLs. In this case, our "app _ name" from above is "homepage" which we combine with "index" into the format "homepage : index". We'll see examples of how this is used later in the tutorial.

  10. Start the server

    With our new urls.py file in place we can start the server again to see our new error message. Run it with the command :

    command line: 
    python manage.py runserver

    The server will dump a big error message that ends with :

    command line error: 
    AttributeError: module 'homepage.views' has no attribute 'index'

    This is exactly what we're expecting since we know that we haven't yet created an "index" function in the homepage app's "views.py" file. We'll do that now.

  11. Add and "index" function to the homepage app's "views.py" file

    The file we need to edit is :

    file path: 
    homepage/views.py

    The default version that was created when we ran the "startapp" command contains only a single line of code and a comment that looks like this :

    original file: homepage/views.py
    from django.shortcuts import render

# Create your views here.

    We can remove the comment and create a new "index" function that Django is looking for. Here's the full contents of what the file should look like :

    replacement for: homepage/views.py
    from django.shortcuts import render

def index(request):
    context = {}
    return render(request, "homepage/index.html", context)

    paste that in and save the file which will automatically reload the server and clear the errors and should show this response :

    command line message: 
    Watching for file changes with StatReloader
Performing system checks...

System check identified no issues (0 silenced).
September 26, 2024 - 22:41:21
Django version 5.1.1, using settings 'tutorial_site.settings'
Starting development server at http://127.0.0.1:8000/
Quit the server with CONTROL-C.

    At this point we can open our browser to our home page which will have changed from the default Django home page to an error page that starts with :

    web browesr error: 
    TemplateDoesNotExist at /

    We'll address that in a moment, but first let's look and the file we just updated.

  12. Examine the homepage views.py file

    Details :

    • The first line is :

      python
      from django.shortcuts import render

      This imports some of the built - in Django functionality for a render() function that we'll use as the return value for our function.

    • The next code line is :

      python
      def index(request):

      This creates a new Python function named "index" that expects a "request" object to be passed to it. Since we're in the "views.py" file this is what the "views.index" portion of the path() function in the homepage "urls.py" file from above points to.

    • Our next line is :

      python
      context = {}

      The context is something that's required to be passed to render() on the next line. We don't need to do anything with it on our homepage so it's empty here.

    • Our last line is :

      python
      return render(request, "homepage/index.html", context)

      This is the respose that's returned to Django for assembly into the content that's sent back to the browser. The value that's returned is from the render() function. It takes three arguments :

      • The "request" object that was passed into the "index" function

      • The template to use for the rendering (in this case "homepage/index.html")

      • And the context (which we defined as empty in the prior line)

  13. Stop the server

    We'll be making another change that Django can't see automatically along with using the command line to create a couple of directories for us. So, stop the server with :

    keyboard: 
    control + c

  14. Create the homepage app templates directory

    The homepage "views.py" file we just updated defined the template the render() function should use as "homepage/index.html". Django automatically looks for template directories inside apps to find those files. When we created the "homepage" app with "startapp" it didn't automatically create a "templates" directory for us. We need to do it ourselves.

    Because of the way namespacing works in Django will also need to create another directory named "homepage" inside it. It's a little weird to look at and we'll discuss why in a moment but for now we need to create a directory tree that looks like :

    directory path: 
    homepage/templates/homepage/

    You can create this directory from the terminal command line with :

    command line: 
    mkdir -p homepage/templates/homepage

  15. Make the home page "index.html" template file

    Now we're ready to drop in our "index.html" template file at :

    file path 
    homepage/templates/homepage/index.html

    Copy and past this for the content of the file and save it.

    new file: homepage/templates/homepage/index.html
    <!DOCTYPE html>
<html>
  <body>

    <h1>Hello, world</h1>

  </body>
</html>

  16. Start the server

    We're ready to start the server again with :

    command line: 
    python manage.py runserver

    Now, we can go back to our site and we'll see our bare bones home page at :

    home page:

    http://127.0.0.1:8000/

    It may not be a lot to look at, but it's live and that's the first step to making more progress.

Wrapping Up
